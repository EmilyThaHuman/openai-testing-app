# Quickstart Guide

This guide will help you set up and run the OpenAI Testing App locally.

## Prerequisites

Before you begin, ensure you have the following installed:
- Node.js (v18 or higher)
- npm (v8 or higher)
- Git

## Installation

1. Clone the repository:
```bash
git clone https://github.com/yourusername/openai-testing-app.git
cd openai-testing-app
```

2. Install dependencies:
```bash
npm install
```

3. Create a `.env` file in the root directory:
```bash
cp .env.example .env
```

4. Configure your environment variables:
```env
VITE_OPENAI_API_KEY=your_openai_api_key
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
VITE_PERPLEXITY_API_KEY=your_perplexity_api_key
```

## Running the App

1. Start the development server:
```bash
npm run dev
```

2. Open your browser and navigate to:
```
http://localhost:5173
```

## Basic Usage

### Chat Interface

1. Navigate to the Chat page
2. Type your message in the input field
3. Press Enter or click the send button
4. View the AI's response in real-time

### AI Assistants

1. Go to the Assistants page
2. Click "Create New Assistant"
3. Configure the assistant's settings
4. Start interacting with your custom assistant

### Image Generation

1. Visit the Images page
2. Enter your image prompt
3. Select generation options
4. Click "Generate" to create your image

### Open Canvas

1. Access the Open Canvas feature
2. Use the flexible workspace for AI experiments
3. Try different AI providers and capabilities

## Troubleshooting

### Common Issues

1. **API Key Issues**
   - Ensure all API keys are properly set in your `.env` file
   - Verify API keys are valid and have sufficient permissions

2. **Build Errors**
   - Run `npm clean-install` to refresh dependencies
   - Check Node.js version compatibility
   - Clear browser cache and local storage

3. **Runtime Errors**
   - Check browser console for error messages
   - Verify all required environment variables are set
   - Ensure Supabase connection is properly configured

### Getting Help

If you encounter issues:
1. Check the [documentation](/introduction)
2. Search existing GitHub issues
3. Create a new issue with detailed information about your problem
